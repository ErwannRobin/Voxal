# Voxal updates — 2026-06-15

## Responsive room UI overhaul

- Non-tiny mode now keeps the room bottom controls fixed at the bottom.
- Tiny embed mode was refined for dense participant display and clearer active-speaker highlighting.

## Embed / tiny behavior controls

- Tiny mode URL flags: `ui=tiny`, `embed=tiny`, `tiny=1|true`, `compact=1|true`.
- Force web join (skip native-app redirection): `forceWeb=1|true|yes` (also `webOnly`, `web`).
- Iframe header-hiding flag: `hideHeader=1|true|yes` (alias: `noHeader`), applied only when embedded.

## Identity and naming

- Duplicate user names are prevented room-wide by host-authoritative canonicalization.
- Anonymous users receive random `Color + Animal` names (with color).
- When a chosen name is already taken, users now get a visible warning toast and an auto-adjusted unique name.

## Presence channel reliability

- Hosts refresh presence session metadata on membership changes (live peer count).
- Presence channel joins try all connected peer IDs before failing.
- Deputy takeover remains joinable for future users after host disconnects through improved fallback/retry behavior.
- Active presence-managed rooms now derive the local username from the presence API `display_name` for the current `peer_id`.

## Auth caller routing

- OAuth connect URLs now carry caller hints so the auth service can send web callers back to the web app and desktop callers back to the desktop app.
- Web auth callbacks can now be relayed through either `postMessage` or a `web.voxal.app` token callback URL.

## URL / room handling

- `?room=` stays in the URL (query params are preserved instead of stripped).
- Tiny-mode and embed flows keep room/share context stable.
